Why It Matters

Business Impact

Unplanned downtime represents one of the most significant operational challenges in modern industry. Predictive maintenance shifts the paradigm from reactive repairs to proactive interventions, optimizing equipment lifespan and operational efficiency.

Reduce Unplanned Downtime

Catch failures before they happen. The model identifies 76.5% of impending failures, giving maintenance teams time to schedule repairs during planned downtime windows instead of emergency stops.

Cut Maintenance Costs

Preventive maintenance costs up to 12× less than emergency repairs. By targeting only high-risk machines, you avoid wasteful blanket inspections and allocate technician time where it matters most.

Trustworthy Alerts

With 70.3% precision, when the model flags a failure, it's worth investigating. Too many false alarms erode trust—this model balances catch rate with credibility so teams actually respond to alerts.

Real-Time Decision Support

Getting Started

How to Use the App

Two simple ways to get predictions—manual input for custom scenarios, or one-click testing from real data.

1

Manual Input

Enter sensor readings directly into the form fields:

Select Product Type — Choose L (Low), M (Medium), or H (High) quality grade
Enter Temperatures — Air temp and process temp in Kelvin (typically 295–305K and 305–315K)
Set Machine Parameters — Rotational speed (RPM), torque (Nm), and tool wear (minutes)
Click "Predict" — View the failure probability, confidence level, and action recommendation instantly
2

Test Data Checkbox

Use real test data with a single click:

Scroll to Test Data Table — Browse 2,000 real sensor readings from the holdout test set
Check Any Row — Click the checkbox on the right side of any row to auto-fill all input fields above with that row's sensor data
Filter by Status — Use the filter icon on the "Machine Failure" column to show only Normal or Failure rows
Compare Predictions — Check if the model's prediction matches the actual failure label shown in the table
Under the Hood

Model & Data Overview

Built on the AI4I 2020 Predictive Maintenance Dataset from UCI Machine Learning Repository.

Dataset

  • 10,000 industrial sensor records
  • ~3.4% failure rate (heavily imbalanced)
  • Features: temperature, RPM, torque, tool wear, product type
  • 80/20 stratified train-test split

Model Selection

  • Compared 7 classifiers (LR, KNN, DT, RF, AdaBoost, GBM, XGBoost)
  • XGBoost selected for best recall-precision balance
  • scale_pos_weight for class imbalance
  • RandomizedSearchCV (30 iter, 5-fold CV, F1 scoring)

Action Levels

  • Normal — <20% failure probability
  • Monitor — 20–39% failure probability
  • Warning — 40–69% failure probability
  • Critical — ≥70% failure probability

Tech Stack

  • Backend: FastAPI + Uvicorn
  • ML: XGBoost, scikit-learn, pandas
  • Frontend: Jinja2, vanilla JS, CSS
  • Deploy: Docker / Docker Compose
